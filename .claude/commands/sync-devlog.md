Write a new dated entry at the bottom of DEVLOG.md for today's session. Follow this structure exactly:

```
## YYYY-MM-DD — <short title summarizing the session's main theme>

**What happened:**
<1–2 sentences of context — what prompted today's work>

**<Topic heading>:**
<detail — decisions made, things tried, things rejected and why>

Repeat topic headings as needed. Each heading covers one meaningful thread.

**Outcome:**
- bullet list of concrete artifacts produced or decisions locked in
```

Rules:

- Date must be today's actual date
- Be extremely concise: one sentence per bullet, one short paragraph per heading — no exhaustive lists
- Only include things discussed or decided in this session — no forward-looking plans unless explicitly committed
- Decisions should include the reasoning, not just the conclusion
- Things tried and rejected are as valuable as things adopted — include them
- Keep it factual and terse; this is a dev journal not a report
- Do not summarize previous entries; only append the new one
