# External Integrations

**Analysis Date:** 2026-05-24

## APIs & External Services

**AI / LLM:**

- Perplexity Sonar API — fetches regional news headlines; one parallel call per region
  - SDK/Client: `ws` (WebSocket) via `cron/src/lib/perplexityClient.ts` (HTTP client with iterative retry)
  - Auth: `PERPLEXITY_API_KEY`
  - Used in: `cron/src/fetchNews.ts` (`PerplexitySource` — implements `DigestSource`)
  - Prompt construction: `cron/src/prompt.ts`

- Anthropic Claude API — per-region headline reordering + cross-region global selection
  - SDK/Client: `@anthropic-ai/sdk ^0.97.0`
  - Auth: `ANTHROPIC_API_KEY`
  - Used in: `cron/src/rankHeadlines.ts`
  - Prompt construction: `cron/src/prompt.ts`

**Currency Rates (app):**

- jsDelivr CDN (primary) + Cloudflare (fallback) — currency exchange rate data
  - Used in: `app/src/hooks/useCurrencyRates.ts`
  - No auth required (public CDN endpoints)
  - Module-level cache; percentage change computation

## Data Storage

**Databases:**

- Supabase (PostgreSQL) — primary data store
  - Three tables: `devices`, `user_preferences`, `digests` (see `supabase/schema.sql`)
  - **`digests`**: one row per region per date; `payload` JSONB `{ headlines: Headline[] }`; evicted after `evictDBDays` days
  - **`devices`**: one row per app install; UUID stable across FCM token rotations; `notify_at` TIME (null = notify at cron time)
  - **`user_preferences`**: one row per authenticated user; `preferences` JSONB; RLS `auth.uid() = user_id`
  - Two-key model:
    - **Publishable key** (`EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY`) — used in `app/` via `app/src/supabase/client.ts`
    - **Secret key** (`SUPABASE_SECRET_KEY`) — used in `cron/` for server-side writes; bypasses RLS
  - Connection (cron): `SUPABASE_URL` + `SUPABASE_SECRET_KEY`
  - Connection (app): `EXPO_PUBLIC_SUPABASE_URL` + `EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
  - Client (cron): `@supabase/supabase-js ^2.105.4` in `cron/src/notify.ts`
  - Client (app): `@supabase/supabase-js` with MMKV-backed session persistence in `app/src/supabase/client.ts`
  - Schema reference only (no migrations tooling): `supabase/schema.sql` — apply via Supabase dashboard SQL editor; safe to re-apply (`IF NOT EXISTS`)

**File Storage:**

- Not used

**Caching:**

- MMKV (`react-native-mmkv`) — on-device cache for `app/` only
  - Session persistence: `app/src/storage/mmkv.ts` (shared MMKV instance)
  - Digest cache (cache-first): `app/src/storage/digests.ts`
  - User preferences: `app/src/storage/preferences.ts` (MMKV + Supabase pull/push; conflict resolution via `updatedAt`)

## Authentication & Identity

**Auth Provider:**

- Supabase Auth — built-in authentication for `app/`
  - Implementation: `app/src/hooks/useSupabaseAuth.ts` — sign in/up/out/reset/delete
  - Session management via Supabase client with MMKV persistence
  - Deep link recovery: `app/src/hooks/useDeepLinkRecovery.ts` handles `pulse://reset-password` PKCE + implicit token exchange
  - RLS enforces `auth.uid() = user_id` on `user_preferences`

**Device Identity:**

- UUID generated by app on first launch; stable across FCM token rotations; resets on reinstall
- Stored in `devices` table; linked to `user_id` after login

## Push Notifications

**FCM (Firebase Cloud Messaging):**

- Firebase Admin SDK (`firebase-admin ^13.0.0`) — server-side dispatch in `cron/`
  - Used in: `cron/src/notify.ts` (`dispatchFcm`, `sendNotifications`)
  - Auth: `FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY`
  - `FIREBASE_PRIVATE_KEY` must use literal `\n` for newlines in env

- React Native Firebase (`@react-native-firebase/messaging`) — client-side in `app/`
  - Used in: `app/src/notifications/register.ts` (`registerNotificationHandlers`, `linkDeviceToUser`)
  - FCM token upserted to `devices` table; refreshed via `onTokenRefresh`
  - Requires `google-services.json` at `app/android/app/google-services.json`
  - Expo Go incompatible — requires Expo dev-client

**Notification Flow:**

1. `cron/api/daily-digest.ts` runs at 05:00 UTC — fetches + persists digests + notifies devices with `notify_at IS NULL`
2. `cron/api/notify.ts` runs every 30 min — notifies devices whose `notify_at` falls in the current 30-minute window

## Webhooks & Callbacks

**Incoming:**

- `cron/api/account.ts` — device registration and account deletion endpoint (called by app)
- `cron/api/daily-digest.ts` — Vercel cron trigger (HTTP GET from Vercel scheduler)
- `cron/api/notify.ts` — Vercel cron trigger (HTTP GET from Vercel scheduler)
- All Vercel handlers validate a `CRON_SECRET` header via `checkCronSecret` from `cron/src/config.ts`

**Outgoing:**

- Perplexity Sonar API — WebSocket/HTTP from `cron/src/lib/perplexityClient.ts`
- Anthropic Claude API — HTTP from `cron/src/rankHeadlines.ts`
- FCM — HTTP via `firebase-admin` from `cron/src/notify.ts`
- jsDelivr / Cloudflare CDN — HTTP from `app/src/hooks/useCurrencyRates.ts`

## Deep Links

**Scheme:** `pulse://`

- `pulse://reset-password` — password reset; handled in `app/src/hooks/useDeepLinkRecovery.ts` via PKCE + implicit token exchange
- Expo `expo-linking` (not RN core `Linking`) used for all deep link handling

## Environment Configuration

**Required env vars — cron/:**

- `PERPLEXITY_API_KEY` — Sonar API key
- `ANTHROPIC_API_KEY` — Anthropic/Claude API key
- `SUPABASE_URL` — Supabase project URL
- `SUPABASE_SECRET_KEY` — Supabase service-role key (bypasses RLS)
- `FIREBASE_PROJECT_ID` — Firebase project ID
- `FIREBASE_CLIENT_EMAIL` — Firebase service account email
- `FIREBASE_PRIVATE_KEY` — Firebase private key (literal `\n` newlines)

**Required env vars — app/:**

- `EXPO_PUBLIC_SUPABASE_URL` — Supabase project URL (public)
- `EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY` — Supabase publishable (anon) key

**Secrets location:**

- `.env` files (git-ignored); examples in `cron/.env.example` and `app/.env.example`
- `cron/src/bootstrap.ts` must be first import in all cron runners to load dotenv
